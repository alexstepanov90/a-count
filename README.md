# A-Count — Simple Accounting Platform

Открытая local-first платформа **управленческого учёта** для малого бизнеса.
Счета на оплату, счета-фактуры, акты выполненных работ, накладные, акты сверки,
учёт оплат, остатков ТМЦ, доходов/расходов и справочный расчёт налогов.

**Все данные хранятся только на устройстве пользователя** (IndexedDB) — без сервера,
облака и передачи данных.

> ⚠️ A-Count — бета-версия и инструмент управленческого учёта. Платформа не заменяет
> первичную документацию и бухгалтерский/налоговый учёт, предусмотренные
> законодательством вашей страны, и не освобождает от уплаты налогов. Все расчёты
> справочные — проверяйте их по своему налоговому законодательству.

## ⬇️ Скачать и установить (для всех, без программирования)

Не нужен ни Node.js, ни терминал — просто скачайте готовое приложение под вашу систему.

### Способ 1 — приложение на компьютер

Откройте раздел **[Releases →](../../releases/latest)** (он же справа на этой странице)
и скачайте архив под вашу систему:

- **macOS** — `A-Count-macOS.zip`
- **Windows** — `A-Count-Windows.zip`

Распакуйте архив — внутри лежит подробная инструкция `ПРОЧТИ-МЕНЯ.txt`. Коротко:

**macOS:** перетащите `A-Count.app` в папку «Программы». При первом запуске —
правой кнопкой по значку → «Открыть» → «Открыть» (это разово, стандартная защита
macOS для приложений без платной подписи Apple).

**Windows:** запустите `Создать ярлык на рабочем столе.bat` — на рабочем столе
появится ярлык «A-Count», дальше открывайте платформу по нему как обычную программу.
Если Windows SmartScreen покажет предупреждение — «Подробнее» → «Выполнить в любом случае».

### Способ 2 — прямо в браузере, без установки

Откройте **[alexstepanov90.github.io/a-count](https://alexstepanov90.github.io/a-count/)** —
работает как есть, ничего скачивать не нужно. Данные всё равно сохраняются только
в вашем браузере на вашем устройстве, а не на сервере.

### Первый запуск

Вас встретит короткая регистрация (имя, e-mail, страна, валюта учёта) и пользовательское
соглашение — прочитайте его, там объясняется, что это бета-версия для управленческого
учёта. Дальше сразу можно заводить контрагентов и выставлять первый документ.

**Резервные копии** — раз в неделю: «Мои реквизиты» → «Скачать резервную копию».
Один JSON-файл — это вся ваша база; им же переносите данные на другое устройство.

## Возможности

- **Контрагенты** — реквизиты (для РК: БИН/ИИН, ИИК, БИК, Кбе), автоподстановка в документы
- **Документы** — счёт на оплату, счёт-фактура, АВР (форма Р-1), накладная (форма З-2);
  исходящие и входящие, автонумерация, «создать на основании», связи между документами
- **Печатные формы А4** — АВР и накладная по официальным бланкам приказа МФ РК № 562,
  сумма прописью, печать в PDF через ⌘P; электронная подпись и печать (сканы) по флажку
- **Оплаты и банк** — счета с остатками, журнал платежей, частичная оплата счетов
- **Дебиторы и кредиторы** — сальдо по каждому контрагенту на главной
- **Склад** — остатки ТМЦ по накладным, контроль достаточности при отгрузке
- **Акт сверки** — двусторонний, с сальдо на начало/конец и оплатами
- **Финансы** — доход/расход (кассовый метод), зарплата, налоги по своим ставкам
  (процент от дохода/прибыли или суммы вручную)
- **Международность** — любая валюта и любые ставки НДС
- **Резервная копия** — вся база одним JSON-файлом

## 🛠 Для разработчиков

Нужен [Node.js](https://nodejs.org) LTS.

```bash
npm install
npm run dev      # разработка: http://localhost:5173
npm run build    # сборка в один файл dist/index.html
```

Сборка — **один самодостаточный HTML-файл**: его можно открыть двойным кликом,
положить на флешку или на любой статический хостинг. Стек: React + TypeScript,
Dexie.js (IndexedDB) для хранения, Vite со сборкой в single-file.

### Онлайн-версия (GitHub Pages)

В репозитории настроен workflow `.github/workflows/deploy.yml`: при каждом пуше в
`main` платформа автоматически публикуется на GitHub Pages. Включается один раз:
**Settings → Pages → Source: GitHub Actions**.

Важно: даже размещённая в интернете, платформа остаётся local-first — сервер отдаёт
только код, данные каждого пользователя живут в его браузере.

### Как выпустить новую версию в Releases

1. Соберите проект: `npm run build`.
2. Обновите файлом `dist/index.html` содержимое пакетов `A-Count-macOS.zip` и
   `A-Count-Windows.zip` (внутри `.app/Contents/Resources/` и рядом с `.bat`-файлами).
3. **Releases → Draft a new release** → новый тег (например `v0.5.0`) → перетащите
   оба архива в область вложений → **Publish release**.

### Десктоп (Tauri) и мобильные (Capacitor)

```bash
# macOS/Windows — нативная сборка через Tauri (вместо ручного .app/.bat из Releases)
npm i -D @tauri-apps/cli && npx tauri init   # web assets: ../dist
npx tauri build

# iOS/Android
npm i @capacitor/core @capacitor/cli
npx cap init "A-Count" "app.acount" --web-dir=dist
npx cap add ios && npx cap add android
```
